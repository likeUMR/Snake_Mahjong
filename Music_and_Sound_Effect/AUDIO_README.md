# 音频资源命名规范 (Audio Asset Naming Convention)

本项目音频系统支持灵活的文件格式（推荐 `.mp3`），请按照以下结构和命名方式放置资源。

## 1. 背景音乐 (Background Music)
**路径**: `Music_and_Sound_Effect/Background_Music/`
- 支持顺序循环播放。
- 默认文件名（可在 `Game.js` 中配置）: `game_master.mp3`, `game.mp3`。
- 支持淡入淡出效果（在 `config.js` 中配置时间）。

## 2. 角色语音 (Character Voices)
**根路径**: `Music_and_Sound_Effect/Voice/`

每个角色拥有独立的文件夹，文件夹名称必须与 `config.js` 中的 `AUDIO_ROLES` 匹配：
- `Player/` (玩家)
- `AI1/` (下家)
- `AI2/` (对家)
- `AI3/` (上家)

### 语音文件名规范 (固定命名):
| 动作 | 文件名 | 触发时机 |
| :--- | :--- | :--- |
| **吃** | `act_chi.mp3` | 成功掠夺产生“吃”时 |
| **碰** | `act_pon.mp3` | 成功掠夺产生“碰”时 |
| **杠** | `act_kan.mp3` | 成功掠夺产生“杠”或自己完成“暗杠”时 |
| **荣和** | `act_ron.mp3` | 通过掠夺对方身体部分的牌导致胡牌时 |
| **自摸** | `act_tumo.mp3` | 通过吃掉地图上的食物导致胡牌时 |

### 通用音效 (存放于根目录 `Music_and_Sound_Effect/`):
| 动作 | 文件名 | 触发时机 |
| :--- | :--- | :--- |
| **进食** | `eat_food.mp3` | 吃到地图上的食物（且未胡牌）时 |
| **生成铁牌** | `fulu.mp3` | 形成吃/碰/杠或暗杠产生铁牌组时 |
| **眩晕** | `xuanyun.mp3` | 蛇发生碰撞进入眩晕状态时 |
| **游戏胜利** | `ron_music.mp3` | 玩家获胜时（自动停止背景音乐并播放） |
| **游戏失败** | `ron_ko.mp3` | 玩家失败时（自动停止背景音乐并播放） |

## 3. 技术说明
- **多队列播放**: 背景音乐和 4 个角色的语音分别处于 5 个独立的播放队列。
- **角色内顺序播放**: 同一个角色的语音如果连续触发（例如快速吃多个食物），会按触发顺序排队播放，不会互相覆盖。
- **跨角色同时播放**: 不同角色的语音可以同时响起，互不干扰。
- **通用音效自动映射**: 系统会自动将进食、生成铁牌、眩晕等音效指向根目录，无需手动复制到每个角色文件夹。
